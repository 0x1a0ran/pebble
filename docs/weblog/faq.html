Frequently Asked Questions

Configuration
  1. I've installed Pebble to my server, but where is my blog data being stored? After logging in to your blog, click the View configuration link to see where your data is being stored. By default, blog data is saved into a blog subdirectory inside your home directory.
  2. I've started Pebble and all of the blog data is being saved into a blog subdirectory inside my home directory. How can I change this? You can change the value of the blog.dir property in the pebble.properties file. See getting started for more information.

Blog entries
  1. I'm setting up a team blog and I'd like all blog entries to be approved before publication. Is this possible? Yes, see Blog entry moderation for more details.

Comments and TrackBacks
  1. My blog has been getting some comment/TrackBack spam recently, what can I do about this? Pebble has several features to fight spam. See Comment and TrackBack Spam for more details.
  2. My blog has just received a lot of spam, how can I clean this up? See Comment and TrackBack Spam for details on the bulk comment and TrackBack management features.

Technical
  1. What technologies does Pebble use? Pebble uses standard J2SE and J2EE technologies such as JSP, servlets, filters, JSP custom tags, JSTL and JAXP.
  2. Can Pebble store content in a relational database? In theory, yes. The peristence tier is designed to be extensible although only the single implementation exists at the moment.
  3. Can Pebble run under Java 5.0? Yes, Pebble has been compiled and tested under Java 5.0.
  4. Can Pebble run under web/application servers that support higher versions of the Servlet and JSP specifications? Yes, newer versions of the specifications are backwards compatible. As an example, Pebble has been sucessfully deployed and tested against Tomcat 5.0 and 5.5.
