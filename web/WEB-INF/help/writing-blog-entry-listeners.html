Writing Blog Entry Listeners

Blog entry listeners are a type of Pebble plugin that allow you to write code that is called whenever a blog entry is added, removed, changed, approved or rejected. To write your own listener, you need to write a Java class that implements the net.sourceforge.pebble.api.listener.blogentry.BlogEntryListener interface as shown below. To gain access to the associated blog entry, use the getBlogEntry() method on the net.sourceforge.pebble.api.listener.blogentry.BlogEntryEvent object that is passed to the callback methods.

An empty implementation is provided by the net.sourceforge.pebble.api.listener.blogentry.BlogEntryListenerSupport class, which you can use as a starting point. 

package net.sourceforge.pebble.listener;

/**
 * Implemented by classes wanting to be notified of blog entry events.
 *
 * @author Simon Brown
 */
public interface BlogEntryListener {

  /**
   * Called when a blog entry has been added.
   *
   * @param event   a BlogEntryEvent instance
   */
  public void blogEntryAdded(BlogEntryEvent event);

  /**
   * Called when a blog entry has been removed.
   *
   * @param event   a BlogEntryEvent instance
   */
  public void blogEntryRemoved(BlogEntryEvent event);

  /**
   * Called when a blog entry has been changed.
   *
   * @param event   a BlogEntryEvent instance
   */
  public void blogEntryChanged(BlogEntryEvent event);

  /**
   * Called when a blog entry has been approved.
   *
   * @param event   a BlogEntryEvent instance
   */
  public void blogEntryApproved(BlogEntryEvent event);

  /**
   * Called when a blog entry has been rejected.
   *
   * @param event   a BlogEntryEvent instance
   */
  public void blogEntryRejected(BlogEntryEvent event);

}
