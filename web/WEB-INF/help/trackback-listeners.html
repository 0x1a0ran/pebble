TrackBack Listeners

TrackBack listeners are a type of Pebble plugin that allows custom code to be called whenever a TrackBack is added or removed. The following TrackBack listeners are included with the Pebble distribution.

net.sourceforge.pebble.event.trackback.EmailNotificationListener Sends a notification e-mail when a new TrackBack is added to the blog owner, additionally asking for approval if the status of the TrackBack is pending.
net.sourceforge.pebble.event.response.MarkPendingListener Marks all new TrackBacks with a status of pending so that they require approval from a blog contributor before being published on the blog.
net.sourceforge.pebble.event.response.IpAddressListener Maintains both a blacklist and a whitelist of IP addresses, performing the following when a new TrackBack is added.
  1. If the IP address of the TrackBack is contained within the blacklist, the TrackBack is marked as pending and the TrackBack's spam score is increased by 1 point.
  2. Otherwise, if the IP address of the TrackBack is contained within the whitelist, the TrackBack is left as-is.
  3. Otherwise, if the IP address of the TrackBack is contained in neither list, the TrackBack is marked as pending and the spam score remains unaffected.
In addition to this, the TrackBack's IP address is removed from the blacklist (if applicable) and added to the whitelist when it is approved. Vice versa for TrackBacks that are rejected.

The comma separated lists of IP addresses are stored under the IpAddressListener.blacklist and IpAddressListener.whitelist plugin properties.
net.sourceforge.pebble.event.response.LinkSpamListener Checks the content of new TrackBacks for the number of links that they contain. If this is greater than the threshold, the TrackBack is marked as pending and the TrackBack's spam score is increased by 1 point.

The default threshold for the number of links is 3, but can be configured by the LinkSpamListener.trackbackThreshold plugin property.
net.sourceforge.pebble.event.response.ContentSpamListener Checks the content of new TrackBacks for specific regular expressions. If the number of occurences is greater than the threshold, the TrackBack is marked as pending and the TrackBack's spam score is increased by 1 point. This is repeated for the title, author and website properties too.

The default threshold for the number of occurences is 0, but can be configured by the ContentSpamListener.threshold plugin property.

The list of regular expressions can be configured using the ContentSpamListener.regexList property.
net.sourceforge.pebble.event.response.SpamScoreListener Checks the spam score of new TrackBacks and if greater than the threshold, the TrackBack is marked as rejected.

The default threshold is 1, but can be configured by the SpamScoreListener.trackbackThreshold plugin property.
net.sourceforge.pebble.event.response.DeleteRejectedListener Deletes new TrackBacks that have been marked as rejected.
net.sourceforge.pebble.event.response.DisableResponseListener Deletes new TrackBacks, effectively disabling the ability for readers to leave TrackBacks.

To configure the TrackBack listeners used by your blog, simply modify the list on the plugins page. By default, no TrackBack listeners are used.
